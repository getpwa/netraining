Fragments in XML:-
You can think of a fragment as a modular section of an activity, which has its own lifecycle, receives its own input events, and which you can add or remove while the activity is running (sort of like a "sub activity" that you can reuse in different activities)

Simply saying, we can say "Fragments as a reuseable UI components".
It has its own lifecycle and process its own events.
It can be added or removed while the activity runs.
Uses of Fragments:
1. Flexible user interface across different screen sizes.
2. Fixed/Scrolling/Swipe tab displays.
3. Dialog boxes.
4. Actionbar customization with the list and tab modes.
Steps for creating Fragments in XML Layout:
1. Create a class that extends fragment.

public class MyFragment extends Fragment
2. Create a separate layout file or java objects that contain the appearance

<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context="demo.arjun.com.fragmentapp.MyFragment"
    android:id="@+id/cdcd12121">

    <!-- TODO: Update blank fragment layout -->
    <TextView
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:text="@string/hello_blank_fragment" />

</FrameLayout>
3. Use the onCreateView method to link its appearance.

public class MyFragment extends Fragment {
public View onCreateView(LayoutInflater inflater, ViewGroup container,
                             Bundle savedInstanceState) {
        // Inflate the layout for this fragment
        return inflater.inflate(R.layout.fragment_my, container, false);
    }
}
4. Attach the fragment inside the Activity through XML

<fragment
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:name="demo.arjun.com.fragmentapp.MyFragment"
        android:id="@+id/myfragment"/>
