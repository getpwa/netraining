Android Application Sandbox:-
Android Applications runs in a SANDBOX.
Sandbox is an isolated area of the Operating System that does not have access to the rest of the system resources unless access permissions are granted by the user whenever an app is installed.
How does Android achieves the concept of SANDBOX?
Each Android Application is assigned a unique Linux USER ID and runs in separate process.

Note:-
By default android takes the package name as the process name.
But if you define process property in application tag in manifest file
android:process="com.example.newprocessname"
then the application will run with this name "com.example.newprocessname".
So several app packages may run as a part of one process.
Can an Android Application share a common userid?
Yes, they can.
Use android:sharedUserId in AndroidManifest.xml file.If this attribute is set to the same value for two or more applications, they will all share the same ID â€” provided that they are also signed by the same certificate.

Note:- Application with the same user ID can access each other's data and, if desired, run in the same process.
Android InterProcess Communications:-
Communication across process boundaries is known as inter process communication.
It can take place when two app share the same common App User ID.
Thus steps of achieving interprocess communication are:-
  1. App2 wants to read data of App1.
  2. Thus App2 will be needing installation information of App1.
  3. Since PackageManager contains various kinds of information related to the application packages that are currently installed in the device. Use getPackageManager of Context Class.
  4. Call ApplicationInfo getApplicationInfo(String packagename,int flags) of package manager to retrieve all the information about app1.
    packagename-
    ↳The full name (i.e. com.google.apps.contacts) of an application.
    flags-
    ↳Additional option flags. Use any combination of GET_META_DATA, GET_SHARED_LIBRARY_FILES, MATCH_SYSTEM_ONLY, MATCH_UNINSTALLED_PACKAGES to modify the data returned.
Mechanisms of Inter Process Communications are:-
  1. Intents
  2. Messenger
  3. ContentResolver
  4. Binder/IBinder
